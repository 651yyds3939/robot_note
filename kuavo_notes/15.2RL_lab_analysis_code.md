## 一、 整体逻辑：一条「仿真 MDP + PPO」流水线

这个项目是 Isaac Lab 的 `ManagerBasedRLEnv`（命令/观测/动作/奖励/终止/事件/课程 分模块管理） 加上 RSL-RL 的 PPO（`OnPolicyRunner`）。数据流可以概括为：

1. **命令 (Commands)**：例如 `base_velocity`，周期性采样期望线速度、角速度/航向。
2. **动作 (Actions)**：策略输出关节目标（这里是 `JointPositionActionCfg`，带 scale），经执行器作用到机器人。
3. **仿真步进**：Isaac 物理 + 传感器（接触力、射线高度等）。
4. **观测 (Observations)**：拼成 policy / critic 向量（Kuavo 在 rough 里对 critic 加了很多项）。
5. **奖励 (Rewards)**：每个 `RewTerm` 对应一个标量项，乘 `weight` 后由 Isaac Lab 的 `RewardManager` 求和为每步总奖励。
6. **终止 (Terminations)**：超时、非法接触、关节限位等结束 episode。
7. **事件 (Events)**：重置位姿/关节、域随机化、间歇推扰等。
8. **课程 (Curriculum)**：例如按跟踪奖励调节地形难度（`mdp/curriculums.py` 里会读 reward 的 episode 累计）。

**强化学习算法侧**：`train.py` 用 `RslRlVecEnvWrapper` 包一层后交给 `OnPolicyRunner`，内部是标准 **on-policy rollout → PPO 更新**（见 `rsl_rl/rsl_rl/runners/on_policy_runner.py`、`algorithms/ppo.py`）。奖励只出现在环境 `step` 的返回值里，算法本身不定义任务奖励。

## 二、「从 0 开始训练」的逻辑

入口是 `scripts/rsl_rl/train.py`：

* 先 `AppLauncher` 拉起 Isaac Sim；
* `import ext_template.tasks` 完成 Gym 环境注册（任务名 → `env_cfg_entry_point` + `rsl_rl_cfg_entry_point`）；
* Hydra 按 `--task` 载入对应 `EnvCfg` 和 PPO 的 `RunnerCfg`；
* `gym.make(args_cli.task, cfg=env_cfg)` 创建环境；
* `runner = OnPolicyRunner(...)`，默认随机初始化策略网络；
* `runner.learn(..., init_at_random_ep_len=True)` 开始采样与更新。

只有在 agent 配置里打开 `resume` 并指定 `load_run` / `load_checkpoint` 时，才会 `runner.load(resume_path)` 从 checkpoint 接着训；否则就是「从零」的随机策略 + 环境交互。

**任务注册示例 (Flat / Rough Kuavo)** 在 `__init__.py` Lines 13-40：

```python
gym.register(
    id="Legged-Isaac-Velocity-Flat-Kuavo-S42-v0",
    entry_point="omni.isaac.lab.envs:ManagerBasedRLEnv",
    disable_env_checker=True,
    kwargs={
        "env_cfg_entry_point": f"{__name__}.flat_env_cfg:KuavoS42FlatEnvCfg",
        "rsl_rl_cfg_entry_point": f"{agents.__name__}.rsl_rl_ppo_cfg:KuavoS42FlatPPORunnerCfg",
    },
)
# ... Rough 任务指向 rough_env_cfg:KuavoS42RoughEnvCfg ...

```

### 2. 核心配置文件（粗糙地形）: `rough_env_cfg.py` 深度拆解

* **路径**：`exts/ext_template/ext_template/tasks/locomotion/velocity/config/s42/rough_env_cfg.py`
* **说明**：决定「用哪些项、权重多少、传什么参数」的配置（比 `rewards.py` 更常改）。这是实际训练 Rough 任务时最核心的配置面。

#### 💡 补充：`rough_env_cfg.py` 中其他类的大致作用（暂时无需深入细节）

除了你重点攻克的两个核心类外，该文件内其他类属于环境的基础配套（“导演组”），其核心职责如下：

* **`MySceneCfg`（搭建片场）**：配置仿真世界的基础物理实体，包括地形类型、生成器、机器人的导入路径、以及雷达和足端力传感器等硬件的挂载。
* **`CommandsCfg`（动作指标）**：定义给机器人下发的控制命令范围，规定了考官随机发出的前后速度、左右速度和转向角速度的最大与最小区间。
* **`ObservationsCfg`（感官网络）**：定义哪些数据输入网络。分为 Policy（网络只能看到基础关节和IMU数据）和 Critic（裁判拥有上帝视角，能看到高度扫描、真实速度和扭矩等）。
* **`TerminationsCfg`（不及格红线）**：定义Episode提前终止（死亡重置）的条件，比如超时、躯干（Base）非法触地或关节角度彻底越界。

#### 📌 重点攻克类 1：`class RewardsCfg` 详细拆解

在人形机器人强化学习中，`RewardsCfg` 扮演着“指挥棒”和“考核标准”的角色。它不仅决定了机器人要完成什么任务（正奖励），还决定了机器人需要遵守哪些物理限制、保护哪些硬件免受磨损（负奖励/惩罚）。以下将 21 个奖励项按照五个维度进行分类拆解：

##### 1) 任务核心奖励（Core Task Rewards）

促使机器人运动的根本动力，采用**指数核函数（Exponential Kernel）**。特点是：当误差较大时奖励极小，当机器人极度接近目标指令时奖励呈指数级放大，这能逼着机器人精准控速。

* **`track_lin_vel_xy_exp`**
* **奖励函数**：`mdp.track_lin_vel_xy_yaw_frame_exp`（权重：`1.0`）
* **具体数学公式**：
$$R = \exp\left(-\frac{\|\mathbf{v}_{xy} - \mathbf{v}_{xy}^*\|^2}{\sigma^2}\right)$$


 其中 $\mathbf{v}_{xy} = [v_x, v_y]^T$ 为机器人当前在机身航向坐标系（Yaw Frame）下的线速度，$\mathbf{v}_{xy}^*$ 为目标线速度指令向量，$\sigma$ 为控制衰减速率的缩放因子标准差（对应配置中的 `std`）。
* **影响方面**：**前后及左右移动的线速度跟踪。** 它促使机器人跟上发出的 $X$ 轴（前后）和 $Y$ 轴（左右）速度指令。这是让机器人跑起来、停得下的最核心奖励。


* **`track_ang_vel_z_exp`**
* **奖励函数**：`mdp.track_ang_vel_z_world_exp`（权重：`0.5`）
* **具体数学公式**：
$$R = \exp\left(-\frac{(\omega_z - \omega_z^*)^2}{\sigma^2}\right)$$


 其中 $\omega_z$ 为机器人当前绕世界坐标系 $Z$ 轴的偏航角速度，$\omega_z^*$ 为目标偏航角速度指令，$\sigma$ 为标准差。
* **影响方面**：**绕 $Z$ 轴旋转的角速度（航向角速度）跟踪。** 决定了机器人能否听从指令灵活地原地转圈或在前进中转向。



##### 2) 姿态与稳定性惩罚（Posture & Stability Penalties）

人形机器人由于重心高、双足支撑面小，极易晃动。这部分惩罚项用于维持“军姿”，让机器人的躯干保持优雅和稳定。

* **`lin_vel_z_l2`**
* **奖励函数**：`mdp.lin_vel_z_l2`（权重：`-0.2`）
* **具体数学公式**：
$$R = -v_z^2$$


 其中 $v_z$ 为机身质心在垂直世界坐标系 $Z$ 轴方向的线速度分量。
* **影响方面**：**惩罚 $Z$ 轴（垂直方向）的剧烈抖动。** 防止机器人走路时像僵尸一样上下乱跳，让质心运动在水平方向更平滑，极大地节省能量。


* **`ang_vel_xy_l2`**
* **奖励函数**：`mdp.ang_vel_xy_l2`（权重：`-0.05`）
* **具体数学公式**：
$$R = -(\omega_x^2 + \omega_y^2)$$


 其中 $\omega_x, \omega_y$ 分别为机身在基座坐标系（Base Frame）下的横滚（Roll）和俯仰（Pitch）方向角速度。
* **影响方面**：**惩罚横滚（Roll）和俯仰（Pitch）方向的角速度。** 防止机器人的上半身向前栽倒或左右疯狂摇晃，保持躯干稳定。


* **`flat_orientation_l2`**
* **奖励函数**：`mdp.flat_orientation_l2`（权重：`-1.25`）
* **具体数学公式**：
$$R = -\|\mathbf{g}_{xy}\|^2 = -(g_x^2 + g_y^2)$$


 其中 $\mathbf{g}_{xy} = [g_x, g_y]^T$ 是世界重力向量 $\mathbf{g}_{\text{world}} = [0, 0, -1]^T$ 投影到机器人机身坐标系 $X-Y$ 平面上的分量。
* **影响方面**：**惩罚躯干偏离水平面的姿态。** 强制机器人的机身基座（Base）与重力线保持平行。权重高达 `-1.25`，说明保持上半身“挺拔直立”对人形机器人至关重要。


* **`stand_still_without_cmd`**
* **奖励函数**：`mdp.stand_still_without_cmd`（权重：`-0.2`）
* **具体数学公式**：
$$R = \begin{cases} -\sum_{i=1}^{N} \dot{q}_i^2 & \text{if } \|\mathbf{v}_{xy}^*\|_2 + |\omega_z^*| < \epsilon \\ 0 & \text{otherwise} \end{cases}$$


 其中 $\dot{q}_i$ 为机器人各个主动关节的当前角速度，$N$ 为关节总数，$\epsilon$ 为判定速度指令为零的极小阈值。
* **影响方面**：**惩罚零速度指令下的乱动。** 当没有速度指令（目标速度为 0）时，机器人应该乖乖站着。如果不加这一项，机器人为了刷其他生存奖励，可能会在原地持续踩碎步或晃动。


* **`gravity_aligned_when_stopping`**
* **奖励函数**：`mdp.gravity_aligned_when_stopping`（权重：`0.1`）
* **具体数学公式**：
$$R = \begin{cases} 1.0 - \|\mathbf{g}_{xy}\|^2 & \text{if } \|\mathbf{v}_{xy}^*\|_2 + |\omega_z^*| < \epsilon \\ 0 & \text{otherwise} \end{cases}$$


 其中 $\mathbf{g}_{xy}$ 为重力向量在机身坐标系 $X-Y$ 平面的投影向量，$\epsilon$ 为零速度判定的极小阈值。
* **影响方面**：**停止时的重力对齐正奖励。** 当收到停止指令时，若身体完美直立则给予奖励，帮助机器人从运动状态平稳地过渡到静止站立。



##### 3) 关节与动作平滑性惩罚（Hardware Protection & Sim2Real）

在仿真中，算法可能会输出一堆频率极高、扭矩极大的“鬼畜”动作。这部分惩罚是 **Sim2Real（仿真到真实世界迁移）的生命线**。如果权重没调好，训练出来的策略一上真机，电机就会立刻过热烧毁。

* **`dof_acc_l2`**
* **奖励函数**：`mdp.joint_acc_l2`（权重：`-2.5e-7`）
* **具体数学公式**：
$$R = -\|\ddot{\mathbf{q}}\|^2 = -\sum_{i=1}^{N} \ddot{q}_i^2$$


 其中 $\ddot{q}_i$ 为主动关节 $i$ 的当前关节角加速度。
* **影响方面**：**惩罚过大的关节加速度。** 过滤掉高频的剧烈振动，让关节电机的动作过渡更平缓。


* **`dof_power_l2`**
* **奖励函数**：`mdp.joint_power_l2`（权重：`-2.0e-5`）
* **具体数学公式**：
$$R = -\sum_{i=1}^{N} P_i^2 = -\sum_{i=1}^{N} (\tau_i \cdot \dot{q}_i)^2$$


 其中 $\tau_i$ 为关节 $i$ 输出的瞬时电动扭矩，$\dot{q}_i$ 为关节 $i$ 的当前角速度，$P_i$ 即为该关节的瞬时机械功耗。
* **影响方面**：**惩罚瞬间的机械功耗（$扭矩 \times 速度$）。** 贯彻“节能减排”方针，不让机器人做无用功，防止多个关节之间互相“对抗”消耗能量。


* **`dof_torques_l2` 与 `dof_torques_ankle_l2**`
* **奖励函数**：`mdp.joint_torques_l2`（权重皆为：`-1.0e-5`）
* **具体数学公式**：
$$R = -\|\boldsymbol{\tau}_{\text{selected}}\|^2 = -\sum_{i \in \mathcal{S}} \tau_i^2$$


 其中 $\mathcal{S}$ 为所选定的关节集合（前者针对腿部 1-5 号关节和手臂，后者专门指定针对 6 号关节），$\tau_i$ 为该关节输出的关节扭矩。
* **影响方面**：**惩罚过大的关节扭矩。** 代码中将其分为了两组，一组针对腿部 1-5 号关节和手臂，另一组专门针对 6 号关节（**踝关节**）。踝关节在平衡控制中承受着高频冲击，单独拿出来微调有利于精细化保护脚踝。


* **`action_rate_l2` 与 `action_smoothness_l2**`
* **奖励函数**：`mdp.action_rate_l2`（`-0.005`） / `mdp.action_smoothness_l2`（`-0.01`）
* **具体数学公式**：
对于 `action_rate_l2`（动作一阶导数惩罚）：
$$R_{\text{rate}} = -\|\mathbf{a}_t - \mathbf{a}_{t-1}\|^2$$



对于 `action_smoothness_l2`（动作二阶导数惩罚）：
$$R_{\text{smooth}} = -\|\mathbf{a}_t - 2\mathbf{a}_{t-1} + \mathbf{a}_{t-2}\|^2$$



其中 $\mathbf{a}_t$ 为当前控制周期（Time-step $t$）网络输出的关节目标位置动作向量，$\mathbf{a}_{t-1}$ 和 $\mathbf{a}_{t-2}$ 为前一周期和前前一周期的动作向量。
* **影响方面**：**惩罚前后控制周期之间动作（Action）的突变。** 这是削减命令抖动的最强手段，确保策略输出的控制曲线极其丝滑，可以直接被真机驱动器执行。


* **`joint_deviation_hip` 与 `joint_deviation_arms**`
* **奖励函数**：`mdp.joint_deviation_l1`（权重皆为：`-0.1`）
* **具体数学公式**：
$$R = -\|\mathbf{q}_{\text{selected}} - \mathbf{q}_{\text{selected, default}}\|_1 = -\sum_{i \in \mathcal{S}} |q_i - q_{i, \text{default}}|$$


 其中 $q_i$ 为关节当前位置（弧度值），$q_{i, \text{default}}$ 为该关节指定的初始默认（名义）位置角度，$\mathcal{S}$ 为所选的特定关节集合（如髋关节或双臂关节）。
* **影响方面**：**惩罚关节偏离初始默认姿态。** 采用 $L_1$ 范数惩罚。这里分别限制了髋关节（Hip）和手臂（Arms）。这能防止机器人走路时“胯骨轴子拧出奇怪的角度”，同时让双臂自然下垂或保持标准准备姿势，避免手臂像风车一样乱甩。



##### 4) 步态与足端控制（Gait & Foot Control）

双足机器人想走得像人，核心在于双脚的交替配合。

* **`feet_air_time`**
* **奖励函数**：`mdp.feet_air_time_clip`（权重：`10.0`）
* **具体数学公式**：对于每一只刚刚从离地状态转换为与地面接触（发生首次触地 Contact Event）的脚 $i$，其获得的单项累加奖励为：
$$R_i = \min(t_{\text{air}, i} - t_{\text{min}}, t_{\text{max}} - t_{\text{min}}) \cdot \mathbb{I}(\text{contact}_i \text{ triggers})$$


 总奖励为双足求和。代码中设置的截断区间为 $t_{\text{min}} = 0.2\text{s}, t_{\text{max}} = 0.5\text{s}$。
* **影响方面**：**鼓励合理的滞空时间。** 这是教机器人学会“踏步/倒脚”的关键。如果滞空时间在 0.2 秒到 0.5 秒之间，会获得多达 `10.0` 极其丰厚的正奖励。没有这一项，机器人可能会选择擦着地面滑行（拖步走）。


* **`feet_slide`**
* **奖励函数**：`mdp.feet_slide`（权重：`-0.1`）
* **具体数学公式**：
$$R = -\sum_{i \in \text{feet}} \|\mathbf{v}_{\text{foot}, i, xy}\|^2 \cdot \mathbb{I}(\text{contact}_i)$$


 其中 $\mathbf{v}_{\text{foot}, i, xy} = [v_{x,i}, v_{y,i}]^T$ 为脚掌 $i$ 质心在水平面上的绝对线性速度，$\mathbb{I}(\text{contact}_i)$ 是足端接触地面的状态指示函数（触地状态下为 1，在空中离地状态下为 0）。
* **影响方面**：**惩罚足端在支撑相时的滑动（Slip）。** 人类走路时踩实了就不该动。如果机器人在仿真里习惯了“滑冰式”走路，真机在摩擦力不同的地面上就会立刻滑倒。


* **`contact_force`**
* **奖励函数**：`mdp.contact_forces`（权重：`-0.001`）
* **具体数学公式**：
$$R = -\sum_{i \in \text{feet}} \max(0, \|\mathbf{F}_{c, i}\|_2 - F_{\text{threshold}})^2$$


 其中 $\mathbf{F}_{c, i}$ 为足端传感器测得的地面法向及切向三维接触总合力，硬限位阈值 $F_{\text{threshold}} = 900\text{N}$。
* **影响方面**：**惩罚过大的跺脚力。** 设置了 900N 的硬阈值，一旦脚掌砸向地面的力超过这个值就会被罚。这能让机器人的步态更加轻盈，保护真机的足端传感器和机械结构。



##### 5) 约束与生存惩罚（Constraints & Survival）

这部分定义了什么是“不可触碰的红线”。

* **`termination_penalty`**
* **奖励函数**：`mdp.is_terminated`（权重：`-200.0`）
* **具体数学公式**：
$$R = \begin{cases} -1.0 & \text{if environment triggers termination} \\ 0.0 & \text{otherwise} \end{cases}$$


 （该惩罚值与当前帧的其余奖励叠加，发生提前重置时乘以权重即直接重扣 200 分）。
* **影响方面**：**提早摔倒的超级巨额惩罚。** 一旦机器人摔倒（触发了 `TerminationsCfg` 里的条件，如机身着地或关节限位违法导致 Episode 提前结束），直接重罚 200 分。这是策略首先要去规避的最高代价，逼着机器人快速学会“活着最重要”。


* **`undesired_contacts`**
* **奖励函数**：`mdp.undesired_contacts`（权重：`-1.0`）
* **具体数学公式**：
$$R = -\sum_{j \in \mathcal{B}_{\text{undesired}}} \mathbb{I}(\|\mathbf{F}_{c, j}\|_2 > F_{\text{limit}})$$


 其中 $\mathcal{B}_{\text{undesired}}$ 为不允许发生地面碰撞接触的刚体集合（如大小腿连杆、躯干连杆、手臂等），触力判定硬门槛阈值 $F_{\text{limit}} = 1.0\text{N}$。
* **影响方面**：**非正常接触惩罚。** 限制了小腿以上部位、机身和手臂触地。人形机器人走路只能脚掌着地，如果膝盖、屁股或手臂碰到了地面或障碍物，说明姿态已经完全失控。


* **`dof_pos_limits`**
* **奖励函数**：`mdp.joint_pos_limits`（权重：`-1.0`）
* **具体数学公式**：
$$R = -\sum_{i=1}^{N} \left( \max(0, q_i - q_{i, \text{max\_safe}})^2 + \max(0, q_{i, \text{min\_safe}} - q_i)^2 \right)$$


 其中 $q_i$ 为关节当前的角度位置，$q_{i, \text{max\_safe}} = q_{i, \text{max}} - \delta$ 且 $q_{i, \text{min\_safe}} = q_{i, \text{min}} + \delta$ 为设定的软安全壁障边界（$\delta$ 通常为接近软限位的缓冲区小角度）。
* **影响方面**：**惩罚关节接近物理限位（Limit）。** 避免机器人为了保持平衡强行把关节掰到极限死角（造成机械卡死或损坏），引导关节在舒适的中间安全范围内运动。



##### 💡 总结奖励权重设计特点

纵观整个 `RewardsCfg`，我们可以发现一个明显的金字塔设计：

1. **最高优先级**是生存（`termination_penalty = -200`）与步态引导（`feet_air_time = 10`），先确保能交替迈步且不摔倒；
2. **第二优先级**是躯干姿态稳定（`flat_orientation_l2 = -1.25`）和速度跟踪（`track_lin_vel_xy_exp = 1.0`），让它站得直且走得准；
3. **底层基石**是微小但密集的硬件平滑惩罚（如 `1.0e-5`、`2.5e-7` 级别），用来细细打磨动作的平滑度，防止出现毁坏硬件的震颤。

#### 📌 重点攻克类 2：`class EventCfg` 详细拆解

它的核心功能是实现 **域随机化（Domain Randomization, DR）** 和 **环境重置（Reset）**。仿真环境里的物理参数（如重力、摩擦力、质量）往往太过于完美，如果机器人只在完美的仿真世界里训练，一到真实世界（Sim2Real）就会因为微小的物理差异（比如地面滑了一点、电池重了一点）而直接摔倒。`EventCfg` 通过在不同阶段动态修改物理世界和机器人的参数，逼着机器人练就一身“以不变应万变”的鲁棒（Robust）本领。

##### 📊 EventCfg 全景速查表

| 事件名称 (Event Name) | 触发时机 (Mode) | 随机化/重置的核心对象 | 惩罚与训练目的 |
| --- | --- | --- | --- |
| `physics_material` | **startup** (启动时) | 地面与机器人的摩擦力、弹性 | 适应摩擦力不同的地面（如瓷砖、地毯、泥地） |
| `add_base_mass` | **startup** (启动时) | 躯干（Base）的附加质量 | 适应由于加装传感器或电池导致的体重变化 |
| `scale_link_mass` | **startup** (启动时) | 四肢（手臂和腿部）的质量缩放 | 应对机械臂、腿部零部件制造时的物理公差 |
| `randomize_rigid_body_com` | **startup** (启动时) | 躯干的**重心位置 (CoM)** | 极为关键！防止重心偏移导致机器人向前/后栽倒 |
| `scale_actuator_gains` | **startup** (启动时) | 电机控制增益（刚度和阻尼） | 模拟真实电机由于发热、老化导致的力矩输出偏差 |
| `scale_joint_parameters` | **startup** (启动时) | 关节内部摩擦力和转子惯量 | 模拟减速器磨损、润滑油粘度变化导致的关节阻力 |
| `reset_base` | **reset** (重置时) | 机器人复活时的位置与初始速度 | 防止机器人死记硬背出生点，学会从各种动态中开局 |
| `reset_robot_joints` | **reset** (重置时) | 机器人复活时的关节角度 | 让机器人学会从各种奇葩的“初始姿势”中恢复平衡 |
| `base_external_force_torque` | **interval** (进行中) | 周期性给躯干施加瞬间推力 | **著名的“盲推测试”**，训练机器人的抗冲击能力 |

##### 🔍 核心事件深度拆解

###### 1) 初始化阶段 (mode="startup")：抹平仿真与现实的物理鸿沟

这类事件**只在仿真环境刚刚启动时执行一次**。由于训练有 4096 个平行环境（`num_envs=4096`），这意味着系统会并行创造出 4096 个“物理法则微调过”的平行宇宙。

* **`physics_material`（摩擦力与弹性随机化）**
* **参数**：静摩擦力与动摩擦力范围 `(0.0, 2.0)`，恢复系数（弹性）范围 `(0.0, 1.0)`。
* **大白话**：第 1 个环境的地面像溜冰场一样滑（摩擦力 0.1），第 2 个环境的地面像粗糙的水泥地（摩擦力 1.8）。这能让 Kuavo 机器人不管踩在什么材质的地面上，脚底都不会打滑摔倒。


* **`add_base_mass` 与 `scale_link_mass`（体重与肢体重量随机化）**
* **参数**：躯干随机增加或减少 `-5.0kg` 到 `5.0kg`；四肢（`leg_.*_link`, `zarm_.*_link`）的质量按 `0.8` 到 `1.2` 倍进行缩放。
* **大白话**：真实机器人可能今天装了个雷达，明天换了个重一点的电池。通过这个随机化，算法不再依赖精确的机器人数学模型，即便身体重了或轻了，也能靠算法自己把力补回来。


* **`randomize_rigid_body_com`（重心位置随机化）**
* **参数**：$X$、$Y$、$Z$ 三个轴向的重心位置允许有 `(-0.1, 0.1)` 米（10 厘米）的偏移。
* **大白话**：**这是人形机器人训练的重中之重。** 双足机器人对重心（Center of Mass）极度敏感。如果重心向前偏了 5cm，机器人如果不把身体往后倾斜就会前摔。这项训练逼着神经网络学会通过倾角传感器（IMU）实时感知重心的异常，并自主微调姿态。


* **`scale_actuator_gains` 与 `scale_joint_parameters`（电机与关节控制参数随机化）**
* **参数**：电机刚度（Stiffness）和阻尼（Damping）在 `0.8` 到 `1.2` 倍间波动；关节转子惯量（Armature）在 `0.5` 到 `1.5` 倍间波动。
* **大白话**：仿真里的电机模型完美无瑕，但真机的电机发热后输出会变软（增益下降），减速器用久了会有旷量。这能防止网络输出过于激进的电流指令，让控制算法对电机的物理瑕疵有极强的容忍度。



###### 2) 重置阶段 (mode="reset")：走出舒适区，学会死里逃生

当机器人摔倒或者超时（触发 `TerminationsCfg`），当前环境就会重置。重置时，如果每次都把机器人整整齐齐地摆在原点，它就只能学会从“标准完美姿态”开始走路。

* **`reset_base`（随机出生点与初始状态）**
* **参数**：水平位置 $X, Y$ 在 `(-0.7, 0.7)` 米内随机，朝向（Yaw）在 $\pm 180^\circ$ 内随机；**更关键的是**，复活时会自带一个 $\pm 0.3 \text{ m/s}$ 的随机初始线速度和角速度。
* **大白话**：机器人一出生，不仅位置和方向变了，甚至还可能处于“正在被人推着往前冲”或者“身体正在侧倾”的动态瞬间。它必须在复活后的第一帧立刻做出正确反应，极大地增强了接管动态控制的能力。


* **`reset_robot_joints`（随机关节角度复活）**
* **参数**：关节角度在默认姿态的 `0.5` 到 `1.5` 倍之间随机。
* **大白话**：让机器人复活时呈现出轻微的“罗圈腿”或“歪歪扭扭”的姿势，它必须先学会自己把关节掰回正常位置再迈步。



###### 3) 运行阶段 (mode="interval")：冷不丁的黑手（抗冲击训练）

* **`base_external_force_torque`（随机外部推力）**
* **参数**：
* `force_range`: 前后左右推力最高达 $\pm 2500\text{N}$，上下砸力/托力达 $\pm 1500\text{N}$（注意：由于设置了 `probability=0.002`，在大约每 500 个控制步里，机器人会被冷不丁地“猛推一下”）。
* `mode="interval"`, `interval_range_s=(0.0, 0.0)` 代表高频或无固定周期的检查。


* **大白话**：这就是我们在波士顿动力或者各类人形机器人宣传片里最常见到的“员工用曲棍球棍恶意捅机器人”或者“飞来一个大沙袋猛撞机器人”的仿真版本。
* **物理意义**：根据牛顿第二定律 $F = m \cdot \frac{\Delta v}{\Delta t}$，在极短的仿真时间步（如 0.02 秒）施加 2500N 的力，相当于瞬间给机器人施加了一个极大的速度扰动。Kuavo 机器人被推了之后，会触发 `RewardsCfg` 里的各种惩罚，逼着它为了不摔倒，必须立刻“迈出一大步”（Squat 或 Step）来重新撑稳身体。



###### 🛠️ 为什么在 `PLAY`（测试/演示）模式下要关掉它？

在配置文件的最后，有一个 `KuavoS42RoughEnvCfg_PLAY` 类。你会注意到有这样一行：

```python
# remove random pushing event
self.events.base_external_force_torque = None

```

因为 `PLAY` 模式是用来向客户展示成果、或者部署到真机上导出策略（Policy）看的。我们不需要在测试时继续让考官在背后“冷不丁地推机器人一把”了，所以要优雅地把这个推力事件置为 `None`。

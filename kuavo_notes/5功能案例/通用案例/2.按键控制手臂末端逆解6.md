## 1. 描述

- 本文介绍如何使用键盘控制机器人手臂末端位置的功能
- 示例代码位于 `<kuavo-ros-control>/src/manipulation_nodes/motion_capture_ik/scripts/test/keyboard_control_robot_arm_demo.py`
- 支持通过键盘按键实时控制机器人手臂末端执行器的位置和手部开合

没问题！咱们抛开那些冰冷的代码和枯燥的定义，用一个**“公司部门协作”**的通俗比喻，把你刚才跑通的这个 Kuavo 机器人的通信全流程串一遍。

你可以把整个 ROS 系统想象成一家大公司（ROS Master 是公司的总机），而你运行的那些终端和代码，就是公司里的不同部门。

想要让机器人的手臂动起来，公司内部需要经历以下四个标准步骤：

### 第一步：夺取控制权（打内线电话 —— ROS Service）

**部门：** 你的 Python 脚本（大老板） $\rightarrow$ 机器人底层管家

机器人上电后，通常处于“默认待机”或“自平衡”状态（类似自动驾驶）。你作为大老板（Python 脚本），不能上来就瞎指挥，你得先拿回方向盘。
*   **动作：** 你的 Python 脚本调用了 `/change_arm_ctrl_mode` 这个**服务 (Service)**，并发送了参数 `2`[cite: 3]。
*   **大白话：** 大老板给底层管家打了个内线电话：“喂？把手臂的控制权交给我，切换到外部控制模式！”[cite: 3, 4]。管家说：“好的老板，切换成功。”（返回 `result: True`）[cite: 4]。
*   **核心特点：** 服务（Service）就像打电话，是**双向同步**的。你发请求，必须等对方回应你“成功”或“失败”，流程才能继续。

### 第二步：确认当前位置（看 GPS 导航 —— TF 坐标树）

**部门：** 你的 Python 脚本（大老板） $\rightarrow$ 空间定位系统 (TF Buffer)

在命令手臂往前走 0.05 米之前，大老板必须先知道手臂**现在**到底在哪里。
*   **动作：** 脚本通过 `tf2_ros` 库查询 `base_link`（机器人的身体基准）和 `zarm_r7_end_effector`（右手末端）之间的相对位置[cite: 4]。
*   **大白话：** 大老板看了眼公司的 GPS 导航：“哦，原来现在右手在坐标 X: -0.05, Y: -0.25 的位置啊。”[cite: 4]。这是你做任何位移决策的基础。

### 第三步：下达目标指令（发全员广播 —— ROS Topic 发布）

**部门：** 你的 Python 脚本（大老板） $\rightarrow$ C++ 逆解算力部门

大老板按下了键盘上的 `W` 键，意思是“往前走”。大老板不懂怎么控制电机的电流，他只管定目标。
*   **动作：** 脚本将刚才查到的当前 X 坐标加上 0.05（步长），打包成一个 `twoArmHandPoseCmd` 格式的消息，然后发布到 `/ik/two_arm_hand_pose_cmd` 这个**话题 (Topic)** 上[cite: 3, 4]。
*   **大白话：** 大老板拿起大喇叭，在公司走廊里喊了一声（发布话题）：“算力部门听着！我要求右手立马移动到新的坐标 X: 0.00, Y: -0.25，姿态保持不变！”[cite: 4]。
*   **核心特点：** 话题（Topic）就像大喇叭广播，是**单向异步**的。大老板喊完就去干别的事了（继续监听键盘），他不管别人听没听到，也不等别人回话。

### 第四步：翻译与干活（接听广播并执行 —— C++ 节点与 MuJoCo）

**部门：** C++ 逆解算力部门 $\rightarrow$ MuJoCo 仿真车间（或真实电机）

*   **动作 1（翻译）：** 你之前启动的那个 C++ 节点 `arms_ik_node` 一直在监听大喇叭。它听到了目标坐标，立刻掏出复杂的数学公式（运动学逆解 IK），疯狂计算：“为了让手到达 X: 0.00，肩膀的电机需要转 15度，手肘需要转 30度，手腕转 5度……”
*   **动作 2（干活）：** C++ 节点算出了这 7 个关节的精确角度后，再次通过另一个话题（通常是底层的 Joint Command），把这些角度发给 MuJoCo 仿真器。
*   **大白话：** 算力部门把大老板的“宏观目标”，翻译成了底层车间工人能听懂的“具体转动度数”，仿真车间的虚拟电机一通猛转，你在电脑屏幕上就看到机器人的手往前动了一下！

---

**总结一下：**
你今天晚上跑的整个流程，就是**用 Python 脚本扮演大老板，通过打内线电话（Service）拿到权限，通过大喇叭广播（Topic）下达 XYZ 坐标目标。然后 C++ 节点作为打工人默默把坐标翻译成 7 个电机的角度，最后扔给仿真器去表现出来。**

这就是 ROS 通信在实际项目中最经典的运作方式！

## 2. 编译与启动

### 2.1 编译

```bash
cd <kuavo-ros-control>
catkin build motion_capture_ik
```

### 2.2 启动参数

键盘控制节点的可选参数：

- `--hand`: 选择控制的手臂，可选 `left` 或 `right`，默认为 `right`
- `--step`: 移动步长，单位为米，默认值为 0.05

逆解节点的可选参数：

- `control_hand_side`: 控制的手臂选择
  - `0`: 左臂
  - `1`: 右臂
  - `2`: 双臂

### 2.3 启动示例

1. 启动机器人仿真环境：
```bash
cd <kuavo-ros-control>
source devel/setup.bash
roslaunch humanoid_controllers load_kuavo_mujoco_sim.launch
```

2. 启动手臂逆解节点：
```bash
cd <kuavo-ros-control>
source devel/setup.bash
roslaunch motion_capture_ik ik_node.launch control_hand_side:=1
```

3. 启动键盘控制节点：
```bash
cd <kuavo-ros-control>
source devel/setup.bash
rosrun motion_capture_ik keyboard_control_robot_arm_demo.py --hand right --step 0.03
```

## 3. ROS接口

### 3.1 话题

**发布的话题：**

- `/ik/two_arm_hand_pose_cmd`
  - 类型：`motion_capture_ik/twoArmHandPoseCmd`
  - 描述：发布手臂末端 POSE 给 IK

- `/control_robot_hand_position`
  - 类型：`motion_capture_ik/robotHandPosition`
  - 描述：控制手部开合（仅支持实物）

**订阅的话题：**

- `/ik/result`
  - 类型：`motion_capture_ik/twoArmHandPose`
  - 描述：订阅IK求解结果，获取当前手臂位置

### 3.2 服务

- `/change_arm_ctrl_mode`
  - 类型：`kuavo_msgs/changeArmCtrlMode`
  - 描述：切换手臂控制模式
  - 参数：
    - `control_mode`: 控制模式（2表示外部控制模式）

## 4. 使用指南

### 4.1 功能概述

通过键盘按键实时控制机器人手臂末端执行器的位置：
- 位置控制：
  - W/S：前后移动
  - A/D：左右移动
  - Q/E：上下移动
- 手部控制：
  - H：抓取（关闭手掌）
  - K：松开（打开手掌）

### 4.2 主要步骤

1. **初始化**：
   - 启动仿真环境
   - 启动逆解节点
   - 切换到外部控制模式

2. **位置获取**：
   - 尝试从tf获取当前手臂末端位置
   - 如果无法获取则使用默认位置

3. **实时控制**：
   - 通过键盘输入控制手臂位置
   - 发布位姿命令到逆解节点
   - 控制手部开合（仅实物）

### 4.3 注意事项

1. 启动顺序必须按照步骤进行
2. 移动步长建议在0.01-0.05之间
3. 确保目标位置在工作空间内
4. 手部开合功能仅支持实物机器人

## 6. 常见问题

1. **无法获取tf信息**
   - 确保仿真环境已正常启动
   - 检查tf树是否完整
   - 等待几秒钟让tf信息完全发布

2. **手臂不响应控制**
   - 检查是否成功切换到外部控制模式
   - 确认当前选择的手臂是否正确
   - 验证步长是否合适

3. **手臂运动不稳定**
   - 尝试减小步长
   - 检查IK求解参数是否合适
   - 确保目标位置在工作空间内
